GoSniff Terminal Cheat Sheet

The commands you'll actually use, explained like a human

📂 Getting Around (Navigation)
cd ~/Desktop/GoSniff
Go to your GoSniff project folder
cd ..
Go up one folder (back out of current folder)
cd ~
Go to your home folder from anywhere
pwd
Print Working Directory — "where am I right now?"
ls
List files and folders in current location
ls src/components/
List files inside a specific folder
Tip: You can also right-click your GoSniff folder in Finder, go to Services, and choose "New Terminal at Folder" to skip the cd step entirely.
🌿 Git (Version Control)
git status
See what files you've changed (your safety check)
git add .
Stage ALL changed files for a commit
git commit -m "your message"
Save a snapshot with a description of what you did
git push
Send your commits up to GitHub
git pull
Download the latest changes from GitHub
git log --oneline
See a short history of recent commits
Typical save-your-work flow: git status (check what changed) → git add . (stage it) → git commit -m "fixed GPS bug" (save it) → git push (upload it). Think of it like Save → Save As → Upload to Cloud.
📄 File Operations
cp source destination
Copy a file from one place to another
cp ~/Downloads/MapView.js src/components/
Copy a downloaded file into your project
cat filename.js
Print a file's contents in the terminal
grep -n "searchTerm" filename.js
Find a word/phrase in a file (shows line numbers)
mkdir foldername
Make a new folder
Remember: Files downloaded from Claude chat arrive as a zip. If there's a duplicate filename, macOS adds a number like MapView (1).js. To copy those, escape the parentheses: cp ~/Downloads/MapView\ \(1\).js src/components/
🚀 Running GoSniff Locally
npm run dev
Start the GoSniff dev server (view at localhost:3000)
Ctrl + C
Stop the dev server
npm install
Install/reinstall project dependencies
npm install packagename
Add a new package to the project
🤖 Claude Code
claude
Launch Claude Code (run from inside your project folder)
claude --continue
Resume your last Claude Code session
claude --resume
Browse and pick from past sessions
claude --doctor
Run diagnostics if something seems broken
/exit
Leave Claude Code (type this inside a session)
/help
See all available Claude Code commands
Shift + Tab
Toggle Plan Mode (Claude explains before acting)
Best practice: Always cd into ~/Desktop/GoSniff BEFORE running claude, so it automatically reads your project files and matches your existing code patterns.
🌐 Deployment (Vercel)
vercel
Deploy a preview build to Vercel
vercel --prod
Deploy to your live/production site
Note: If you push to GitHub with git push, Vercel usually auto-deploys from there too (depending on your settings). So you may not need to run the vercel command manually.
🆘 When Things Go Sideways
Ctrl + C
Stop whatever is running right now
clear
Clean up a messy terminal screen
git stash
Temporarily hide your changes (like an undo shelf)
git stash pop
Bring back your stashed changes
git checkout -- filename.js
Undo changes to a specific file (revert to last commit)
